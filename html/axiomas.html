Axiomas y proposiciones auxiliares

En esta pagina, y por conveniencia, se listan todos los axiomas y proposiciones auxiliares requeridos para formalizar los invariantes de representación y las funciones de abstracción. Previamente se presentan los renombres de los tipos usados.

Renombres de tipos
  • Node es tupla(child: arreglo[2] de puntero(Node), parent: puntero(Node), color: Color, value: Value)
  • Value es Maybe(value_type)
  • value_type es tupla(clave: Key, significado: Meaning)

El TAD Maybe( ) representa un tipo extendido con un valor nulo. En otras palabras, el TAD Maybe se puede usar para representar los valores de los nodos, donde el nodo cabecera no tiene valor y los nodos internos sí tienen valor. (Esto independientemente de si se implementa con herencia o con un puntero o de otra forma.) Tiene dos observadores:

  • nothing?(x): que indica si x tiene un valor nulo, y
  • data(x): que devuelve el valor de x, suponiendo que no es inválido.

La especificación de este TAD queda como ejercicio (no obligatorio).

Axiomas y proposiciones auxiliares

En esta sección se deben incluir todos los axiomas y proposiciones auxiliares que se usen para describir los invariantes de representación, las funciones de abstracción, las precondiciones y las postcondiciones.

Comentarios
Recordar incluir un alias en el archivo doxyfile a fin de poder referenciar automaticamente a cada axioma desde las otras páginas.

Se muestran algunos ejemplos a continuación.

esDiccionario?

Retorna true si la secuencia representa un diccionario

esDiccionario?: secu(tupla( , )) bool
esDiccionario?(s) sinRepetidos?(primeros(s))

primeros

Proyecta las primeras componentes de una secuencia de pares

primeros: secu(tupla( , )) secu( )
primeros(s) if vacia?(s) then <> else (prim(s)) primeros(fin(s)) fi

raiz

Devuelve un puntero a la raíz del árbol representado por m.

raiz: map puntero(nodo)
raiz(m) m.header->parent

esArbol

Proposición que dice si una estructura representada con punteros a nodo se corresponde efectivamente con un árbol binario finito. La primera proposición asegura que el árbol termina, mientras que la segunda afirma que ningún nodo es hijo de dos nodos distintos (es decir, no hay ciclos).

esArbol: puntero(Node) bool
esArbol(p) true ( k:nat) arbolK(p,k) arbolK(p,k+1) sinRepetidosAB(arbolK(p,k))

arbolK

sinRepetidosAB

Dice si un árbol binario tiene o no elementos repetidos.

sinRepetidosAB: ab( ) bool
sinRepetidosAB(a) if a = nil then true else (perteneceAB(raíz(a),izq(a)) perteneceAB(raíz(a),der(a))) sinRepetidosAB(izq(a)) sinRepetidosAB(der(a)) fi

perteneceAB

Dice si un elemento está o no en un árbol binario.

perteneceAB: ab( ) bool
perteneceAB(e,a) if a = nil then false else raíz(a) = e perteneceAB(e,izq(a)) perteneceAB(e,der(a)) fi

esABBDicc

Dice si el árbol binario que tiene a p como raíz es un ABB sin claves repetidas. Para asegurar esto último, las funciones auxiliares todosMenores y todosMayores hacen comparaciones estrictas.

esABBDicc: puntero(Node) p bool {esArbol(p)}
esABBDicc(p) if p = nullptr then true else todosMenores(p->child[0],p->value.first) todosMayores(p->child[1],p->value.first) esABBDicc(p->child[0]) esABBDicc(p->child[1]) fi

todosMenores

Dice si todas las claves del árbol binario a cuya raíz apunta p son estrictamente menores a e.

todosMenores: puntero(Node) p Key bool {esArbol(p)}
todosMenores(p,e) if p = nullptr then true else p->value.first e todosMenores(p->child[0],e) todosMenores(p->child[1],e) fi

todosMayores

Dice si todas las claves del árbol binario a cuya raíz apunta p son estrictamente mayores a e.

todosMayores: puntero(Node) p Key bool {esArbol(p)}
todosMayores(p,e) if p = nullptr then true else p->value.first e todosMayores(p->child[0],e) todosMayores(p->child[1],e) fi

esRBTree

Dice si el árbol binario a cuya raíz apunta p cumple el invariante de red-black tree (adaptado a nuestra implementación con hojas nullptr): todos los nodos son o rojos o negros, la raíz es negra, todos los nodos rojos tienen sólo hijos negros y todas las ramas desde cualquier nodo interno tienen la misma cantidad de nodos negros.

esRBTree: puntero(Node) p bool {esArbol(p)}
esRBTree(p) p = nullptr (coloresValidos(p) p->color = Black mismaCantNegros(p) rojoImplicaHijosNegros(p) )

coloresValidos

Dice si el árbol binario a cuya raíz apunta p sólo tiene nodos negros y rojos.

coloresValidos: puntero(Node) p bool {esArbol(p)}
coloresValidos(p) if p = nullptr then true else p->color Header coloresValidos(p->child[0]) coloresValidos(p->child[1]) fi

mismaCantNegros

Dice si todas las ramas que salen desde cualquier nodo del árbol binario a cuya raíz apunta tienen la misma cantidad de nodos negros. Observar que, como estamos pidiendo que los hijos de p cumplan este axioma, alcanza con verificar que la cantidad de nodos negros de una rama (en particular la extrema izquierda) es la misma para los dos hijos, y de esa forma aseguramos que se cumple recursivamente mismaCantNegros.

mismaCantNegros: puntero(Node) p bool {esArbol(p)}
mismaCantNegros(p) p = nullptr ( cantNegros(p->child[0]) = cantNegros(p->child[1]) mismaCantNegros(p->child[0]) mismaCantNegros(p->child[1]) )

cantNegros

Devuelve la cantidad de nodos negros de la rama extrema izquierda del árbol binario a cuya raíz apunta p. Por lo explicado anteriormente, esta función sirve para computar mismaCantNegros.

cantNegros: puntero(Node) p nat {esArbol(p)}
cantNegros(p) if p = nullptr then 0 else cantNegros(p->child[0]) (if p->color = Black then 1 else 0 fi ) fi

rojoImplicaHijosNegros

Dice si todos los nodos rojos del árbol binario a cuya raíz apunta p tienen únicamente hijos negros.

rojoImplicaHijosNegros: puntero(Node) p bool {esArbol(p)}
rojoImplicaHijosNegros(p) if p = nullptr then true else p->color = Red ( noEsRojo(p->child[0]) noEsRojo(p->child[1]) ) rojoImplicaHijosNegros(p->child[0]) rojoImplicaHijosNegros(p->child[1]) fi

noEsRojo

Autoexplicativo.

noEsRojo: puntero(Node) bool
noEsRojo(p) p nullptr p->color Red

familiaCorrecta

Función auxiliar de repMap para verificar si el padre de la cabecera es efectivamente la raíz (y viceversa), si el hijo izquierdo de la cabecera es el menor elemento del árbol y si el hijo derecho de la cabecera es el mayor elemento del árbol.

familiaCorrecta: nodo puntero(Node) p bool {p nullptr}
familiaCorrecta(n,p) n.child[0] = getDMost(p,0) n.child[1] = getDMost(p,1) n.parent = p p->parent = n

getDMost

Si d es 0, devuelve el nodo de extrema izquierda del árbol binario a cuya raíz apunta p; si es 1, devuelve el de extrema derecha.

getDMost: puntero(Node) p nat d puntero(Node) {d 1 p nullptr esArbol(p)}
getDMost(p,d) if p->child[d] = nullptr then p else getDMost(p->child[d],d) fi

parentCorrecto

Dice si todos los nodos del árbol binario cuya raíz es apuntada por p tienen como padre al nodo que los tiene como hijos.

parentCorrecto: puntero(Node) p bool {esArbol(p)} parentCorrecto(p) if p = nullptr then true else (p->child[0] nullptr p->child[0]->parent = p) (p->child[1] nullptr p->child[1]->parent = p) parentCorrecto(p->child[0]) parentCorrecto(p->child[0]) fi

cantNodos

Devuelve la cantidad total de nodos del árbol binario a cuya raíz apunta p.

cantNodos: puntero(Node) p nat {esArbol(p)}
cantNodos(p) if p = nullptr then 0 else suc(cantNodos(p->child[0]) + cantNodos(p->child[1])) fi

perteneceClaveABB

Dice si la clave c pertenece al árbol binario cuya raíz es apuntada por p.

perteneceClaveABB: Key puntero(Node) p bool {esArbol(p)}
perteneceClaveABB(c,p) if p = nullptr then false else p->value.first = c perteneceClaveABB(c,p->child[0]) perteneceClaveABB(c,p->child[1]) fi

significadoABB

Devuelve el significado de la clave c en el árbol binario cuya raíz es apuntada por p. Si hay dos tuplas en el árbol que tienen a c como primer componente, devuelve la que esté más a la izquierda, pero en este contexto sólo la llamaremos con árboles que son diccionarios y por lo tanto no tienen repetidos, así que esto no hace diferencia.

significadoABB: Key puntero(Node) p Meaning {esArbol(p) perteneceClaveABB(c,p)}
significadoABB(c,p) if p->value.first = c then p->value.second else ( if perteneceClaveABB(c,p->child[0]) then significadoABB(c,p->child[0]) else significadoABB(c,p->child[1]) fi ) fi

elHeaderEstaPiola

Función auxiliar para verificar la correctitud de cuando el árbol representado es vacío. Responde a la estructura dada por la cátedra, en la que el constructor sin parámetros de map genera un árbol cuya cabecera tiene como hijo izquierdo y derecho a sí misma.

elHeaderEstaPiola: map bool
elHeaderEstaPiola(m) m.header.parent = nullptr m.header.child[0] = &(m.header) m.header.child[0] = &(m.header) m.count = 0

parentK

Función similar a arbolK, pero en vez de devolver los primeros k niveles de un árbol, devuelve el resultado de subir k veces por el nodo apuntado por p. Si se llega a un nodo que tiene como padre a nullptr, devuelve un puntero al último nodo que tenía como padre a un puntero no nulo.

parentK: puntero(Node) p nat puntero(Node) {p nullptr}
parentK(p,k) if p->parent = nullptr then p else (if k = 0 then p else parentK(p->parent,k-1) fi ) fi

inorder

Dado un puntero p que apunta a la raíz de un ABB-diccionario válido, devuelve el recorrido inorder del árbol.

inorder: puntero(nodo) p secu(value) {esABBDicc(p)}
inorder(p) if p = nullptr then vacía else inorder(p->child[0]) && (p->value inorder(p->child[1])) fi

buscarRaiz

Dado un puntero a nodo, busca la raíz del árbol al que pertenece.

buscarRaiz: puntero(nodo) p puntero(nodo) {rep_iter(p) p nullptr}
buscarRaiz(p) if p->color = Header then p->parent else buscarRaiz(p->parent) fi

despuesDe

Dados un elemento e y una secuencia s que lo tiene como primer campo de alguno de sus elementos, devuelve una subsecuencia que tiene como primer elemento a una tupla cuyo primer campo es e y termina al final de s. En caso de haber varias, devuelve la que empieza con la primera aparición.

despuesDe: e secu( ) s secu( ) {esta(e,primeros(s))}
despuesDe(e,s) if prim(s).first = e then s else despuesDe(e,fin(s)) fi

antesDe

Similar a despuesDe, pero devuelve la subsecuencia que termina justo antes de la primera aparición del elemento.

antesDe: Key e secu(value) s secu( ) {esta(e,primeros(s))}
antesDe(e,s) if prim(s).first = e then vacía else prim(s) antesDe(e,fin(s)) fi

longDicc

Como su nombre lo indica, devuelve la longitud de un diccionario.

longDicc: dicc(Key,Meaning) nat
longDicc(d) if vacío?(d) then 0 else suc(longDicc(borrar(dameUno(claves(d)),d))) fi

menoresOrdenados

Dados un diccionario d y una clave c, devuelve una lista ordenada con todas las claves de d que sean estrictamente menores que c y sus respectivas definiciones. No hace falta que c esté definida en d.

menoresOrdenados: Key dicc(Key,Meaning) secu(value)
menoresOrdenados(c,d) if vacío?(d) then vacía else ( if dameUno(claves(d)) c then insertarOrdenado(((dameUno(claves(d)),obtener(dameUno(claves(d)))), menoresOrdenados(borrar(dameUno(claves(d)),d)))) else menoresOrdenados(borrar(dameUno(claves(d)),d)))) fi ) fi

mayoresOrdenados

Similar a menoresOrdenados. En este caso se pide que los elementos sean mayores o iguales que c, ya que estas funciones son usadas para emular los observadores de un iterador que apunta a c.

mayoresOrdenados: Key dicc(Key,Meaning) secu(value)
mayoresOrdenados(c,d) if vacío?(d) then vacía else ( if dameUno(claves(d)) c then insertarOrdenado(((dameUno(claves(d)),obtener(dameUno(claves(d)))), mayoresOrdenados(borrar(dameUno(claves(d)),d)))) else mayoresOrdenados(borrar(dameUno(claves(d)),d)))) fi ) fi

insertarOrdenado

Dadas una tupla (Key,Meaning) e y una secuencia s ordenada crecientemente por el primer campo, devuelve una secuencia ordenada idéntica a s pero con c insertada en el lugar donde corresponde.

insertarOrdenado: value secu(value) s secu(value) {ordenada(s)}
insertarOrdenado(c,s) if vacía?(s) then c s else (if prim(s) c then c s else prim(s) insertarOrdenado(c,fin(s)) fi ) fi

ordenada

Dice si una secuencia está ordenada.

ordenada: secu( ) bool
ordenada(s) if long(s) 1 then true else prim(s) prim(fin(s)) ordenada(fin(s)) fi

valoresOrdenados

Dado un diccionario d, devuelve una secuencia de tuplas con las claves de d y sus respectivos significados ordenadas de manera creciente por el primer campo.

valoresOrdenados: dicc(Key,Meaning) secu(value)
valoresOrdenados(d) if vacío?(d) then vacía else insertarOrdenado( (dameUno(claves(d)),obtener(dameUno(claves(d)),d)), valoresOrdenados(borrar(dameUno(claves(d)),d)) ) fi

Dado un diccionario d, devuelve una secuencia de tuplas con las claves de d y sus respectivos significados ordenadas de manera decreciente por el primer campo.
